# 监控系统功能使用指南

本文档详细介绍了新增的50+监控功能的使用方法、界面操作指南以及系统验证步骤。

## 1. 功能概览

本系统已集成了全方位的监控功能，覆盖以下9大类：

1.  **环境监控 (ENV)**: 温湿度、漏水、烟雾、精密空调等。
2.  **电力监控 (PWR)**: UPS状态、市电质量、PDU负载、电池组健康度等。
3.  **网络监控 (NET)**: 交换机流量、链路状态、带宽使用率、网络延迟等。
4.  **服务器/硬件监控 (HDW)**: CPU/内存利用率、磁盘IO、RAID状态、风扇转速等。
5.  **安防监控 (SEC)**: 门禁状态、视频监控、红外入侵、人员活动等。
6.  **应用/服务监控 (APP)**: 数据库性能、Web服务状态、API响应时间、中间件监控等。
7.  **告警与通知 (ALT)**: 告警策略、多渠道通知（短信/钉钉/企微）、告警历史等。
8.  **容量/资源管理 (CAP)**: 机柜空间、电力容量、制冷容量、端口资源等。
9.  **运维/日志分析 (OPS)**: 变更记录、巡检日志、系统日志、操作审计等。

## 2. 界面操作指南

所有监控功能均集成在机房详情面板中。

### 2.1 进入监控面板
1.  在3D机房视图中，点击任意机房模型。
2.  屏幕右侧将弹出 **机房详情面板**。

### 2.2 查看功能列表
*   **分类筛选**: 面板中部设有分类标签页（如“环境”、“电力”、“网络”等）。点击标签可筛选显示对应类别的监控功能。
*   **功能卡片**: 每个功能以卡片形式展示，包含：
    *   功能名称与代码（如 `1.01 温湿度监控`）
    *   当前状态（正常/告警/注意）
    *   最新指标值（如 `24.5 °C`）
    *   功能描述悬浮提示

### 2.3 功能交互操作
点击功能卡片右侧的 **“操作”** 按钮，展开操作区域：

1.  **写入指标 (模拟数据上报)**
    *   在“指标值”输入框中输入数值（如 `25`）。
    *   点击 **“写入指标”** 按钮。
    *   系统将记录该指标，并立即刷新界面显示的最新值和历史趋势。

2.  **更新状态**
    *   选择 **状态** (正常/注意/告警/活跃/已恢复)。
    *   选择 **严重程度** (信息/警告/严重)。
    *   点击 **“更新状态”** 按钮。
    *   功能卡片的颜色和状态标签将随之变化。

3.  **触发告警**
    *   在“告警内容”输入框填写消息（如 `温度过高报警`）。
    *   点击 **“触发告警”** 按钮。
    *   该告警将立即出现在下方的“告警列表”中。

4.  **多渠道通知 (7.02 多渠道通知集成)**
    *   找到 `7.02 多渠道通知集成` 功能（或在任意功能告警栏填写通道）。
    *   在通道输入框填写：`短信,钉钉,企业微信`。
    *   点击 **“多渠道通知”** 按钮。
    *   系统将模拟发送通知到指定渠道。

### 2.4 查看历史数据
在展开的功能操作区域下方，可以查看：
*   **最近指标**: 显示最近上报的5条指标数据及时间。
*   **最近告警**: 显示该功能最近触发的告警记录。

## 3. 后端与数据验证

### 3.1 数据库验证
系统使用 PostgreSQL 存储所有配置和数据。已预置初始化数据。
您可以使用以下脚本验证数据完整性：

```bash
# 进入项目根目录
cd /Users/xiamuguizhi/code/cockpit_ALL

# 运行数据验证脚本
python3 verify_data.py
```

### 3.2 模拟数据种子
如果需要重置数据，可以使用种子脚本：

```bash
# 重置数据库并写入初始Mock数据
python3 seed_data.py
```

## 4. 扩展开发指南

### 4.1 添加新功能
无需修改代码，只需在数据库 `monitoring_features` 表中添加记录，或通过 API 接口动态添加。

### 4.2 API 接口
后端提供完整的 RESTful API，文档地址（启动后端后访问）：
`http://localhost:8000/docs`

主要接口：
*   `GET /api/features`: 获取功能列表
*   `POST /api/features/{code}/metrics`: 上报指标
*   `POST /api/alerts`: 触发告警

---
**提示**: 目前系统运行在演示/开发模式，所有数据均为本地数据库持久化存储，重启服务数据不会丢失。
